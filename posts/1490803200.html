vlmcsd实战
发布于 2017-03-30 00:00:00 浏览

前言

vlmcsd是一个开源的KMS（Key Management Service，密钥管理服务）模拟器。C语言版本的vlmcsd现在可以运行于Linux（包括Android）、FreeBSD、Solaris、Minix、Mac OS、iOS和Windows上。

我们可以利用vlmcsd学习Windows和Office的KMS激活机制。

关心激活的朋友请直接跳转到Windows篇或Office篇，在本文附录可获取KMS安装序列号。

vlmcsd

以下内容来自vlmcsd的讨论页。

vlmcsd is
· a replacement for Microsoft’s KMS server
· It contains vlmcs, a KMS test client, mainly for debugging purposes, that also can “charge” a genuine KMS server
· designed to run on an always-on or often-on device, e.g. router, NAS Box, …
· intended to help people who lost activation of their legally-owned licenses, e.g. due to a change of hardware (motherboard, CPU, …)

vlmcsd is not
· a one-click activation or crack tool
· intended to activate illegal copies of software (Windows, Office, Project, Visio)

Source and binaries: http://rgho.st/8hXKrr9Dd
Source only: http://rgho.st/8z9BVlkVC

7zip password is 2017

在64位Linux服务器上搭建KMS模拟器只需运行binaries/Linux/intel/static/vlmcsd-x64-musl-static即可，其他环境类似，在此不再赘述。本站也搭建了KMS模拟器（www.wqy.ac.cn:1688）供实验使用。

另外，python版本的KMS模拟器也可在GitHub上找到。

Windows篇

本方法对Windows Vista/7/8/8.1/10有效。

  1. 右键点击“开始菜单”，运行“PowerShell（管理员）”（对于操作系统版本10.0.15063及以上）或“（管理员）”（对于更低的操作系统版本）；
  2. 输入“slmgr /dli”并回车，查看当前许可证信息，包括许可证版本和当前产品密钥的末5位；
  3. 按照许可证版本与本文附录中对应的KMS密钥的末5位对照，若一致则跳至第步；
  4. 输入“slmgr /upk XXXXX”（将XXXXX替换为当前产品密钥的末5位）并回车，卸载当前产品密钥；
  5. 输入“slmgr /ipk XXXXX-XXXXX-XXXXX-XXXXX-XXXXX”（将XXXXX-XXXXX-XXXXX-XXXXX-XXXXX替换为正确的25位KMS密钥）并回车，安装新的产品密钥；
  6. 输入“slmgr /skms <KMS Server>”（将<KMS Server>替换为你的KMS服务器，例如 www.wqy.ac.cn:1688）并回车，设置KMS服务器；
  7. 输入“slmgr /ato”并回车，尝试利用KMS服务器激活；
  8. 输入“slmgr /dli”并回车，查看激活状态（此步骤可选）。
